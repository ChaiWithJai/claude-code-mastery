# Exercise 1: Create Your CLAUDE.md

**Duration**: 5-10 minutes
**Segment**: 1 (Architecture Mind Shift)

---

## Objective

Create a CLAUDE.md file for your project that documents its architecture.

## Instructions

### Step 1: Navigate to Your Project

```bash
cd /path/to/your/project
```

Or create a test project:
```bash
mkdir ~/claude-code-test && cd ~/claude-code-test
git init
```

### Step 2: Create CLAUDE.md

```bash
touch CLAUDE.md
```

### Step 3: Add the Skeleton

Copy this template and customize:

```markdown
# [Your Project Name]

[One sentence: What this project does]

## Architecture

```
[Your directory structure with explanations]
```

## Key Patterns

- [Pattern 1: Why this matters]
- [Pattern 2: Why this matters]
- [Pattern 3: Why this matters]

## Development Workflow

1. [How to run tests]
2. [How to start dev server]
3. [How to deploy]

## What NOT to Do

- [Anti-pattern 1: Why it's bad]
- [Anti-pattern 2: Why it's bad]
```

### Step 4: Fill in Your Details

Answer these questions:
1. What does your project do? (1 sentence)
2. What are the main directories? (3-5)
3. What patterns should always be followed? (2-3)
4. What should never be done? (2-3)

## Verification

- [ ] File exists at project root
- [ ] Has project description
- [ ] Has architecture section
- [ ] Has at least 2 key patterns
- [ ] Has at least 1 anti-pattern
